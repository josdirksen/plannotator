# Spec: Guided Review (v1)

Date: 2026-07-02 ¬∑ Status: draft for iteration
ADR: `adr/decisions/006-guided-review-first-class-feature-20260702-192821.md`
Synthesis: `adr/research/synthesis-guided-review-20260702-195351.md`

## Summary

A `guide` agent-job provider generates a chaptered review of the current changeset ‚Äî
core implementation first, glue last. The guide renders as a screen takeover (file
tree + dock hidden, header + right sidebar intact): ordered sections, each pairing a
prose overview with live, annotatable per-file diffs. Annotations made in the guide
are ordinary `CodeAnnotation`s ‚Äî same state, same Send Feedback payload.

## Data model

`packages/shared/guide.ts` (vendored to Pi via vendor.sh):

```ts
interface GuideDiffRef {
  file: string;          // repo-relative path, must match a DiffFile.path
  lineStart?: number;    // optional highlight hint (new side), not a slicing boundary
  lineEnd?: number;
  note?: string;         // one-line caption for this diff within the section
}

interface GuideSection {
  title: string;         // "Payment Localization Module"
  overview: string;      // markdown prose: what this part is, why it exists
  kind: "core" | "consequence" | "support";  // drives ordering + glue separation
  diffs: GuideDiffRef[]; // 1..n
}

interface CodeGuideOutput {
  title: string;         // from PR when present, else derived from changes
  intent: string;        // 1-2 sentence framing shown under the title
  sections: GuideSection[];   // ordered: core first, support last
  unplacedFiles?: string[];   // changed files the model didn't place (rendered in a
                              // trailing "Everything else" section ‚Äî coverage guarantee)
}

type CodeGuideData = CodeGuideOutput & { reviewed: boolean[] };  // per-section state
```

Server-side validation on ingest (fail-closed, tour precedent):
- zero sections ‚Üí job `failed` (`GUIDE_EMPTY_OUTPUT_ERROR`)
- `diffs[].file` not in the current `DiffFile` set ‚Üí ref discarded entirely
  (a fabricated path never surfaces); `unplacedFiles` only ever contains real
  changed files that the model itself left unplaced or that never landed in
  any section
- every changed file must appear exactly once across sections + unplacedFiles
  (server enforces by diffing the sets, model is instructed but not trusted)

## Server

New module `packages/server/guide/guide-review.ts` (mirrors `tour/tour-review.ts`):
- `GUIDE_REVIEW_PROMPT` ‚Äî system prompt. Core instruction: order sections the way the
  work was reasoned through; separate glue/low-signal changes into `support` sections;
  overviews explain *why*, not line-by-line *what*. Same discipline as
  `TOUR_REVIEW_PROMPT` (no em-dashes/emoji rules etc. where applicable), but guide-toned.
- `GUIDE_SCHEMA_JSON` ‚Äî JSON schema for `CodeGuideOutput`.
- `buildGuideUserMessage` ‚Äî reuses the diff-context framing pattern
  (`agent-review-message.ts` style) + the changed-file list with per-file stats so the
  model plans placement against the real file set.
- `buildGuideClaudeCommand` / `buildGuideCodexCommand` ‚Äî copy tour's flag sets
  (Claude: `--json-schema`, stream-json, read-only allowedTools; Codex:
  `--output-schema` + output file, `--full-auto --ephemeral`).
- `createGuideSession()` ‚Äî in-memory `guideResults: Map<jobId, CodeGuideOutput>` +
  `guideReviewed: Map<jobId, boolean[]>`; `getGuide`, `saveReviewed`, `buildCommand`,
  `onJobComplete` (parsers: stream for Claude, file for Codex).

Integration (existing seams):
- `packages/server/agent-jobs.ts`: add `"guide"` to `SERVER_BUILT_PROVIDERS`;
  capabilities entry `guide: claude || codex` (same rule as tour, agent-jobs.ts:161).
  No log-formatter change needed: the formatter keys off `spawnOptions?.engine ===
  "claude"` (agent-jobs.ts:314), which the guide's buildCommand result satisfies by
  returning `engine`, tour-style.
- `packages/server/review.ts`: `provider === "guide"` branches in `buildCommand` /
  `onJobComplete` beside tour's (review.ts:690-700, 901-914); routes
  `GET /api/guide/:jobId` and `PUT /api/guide/:jobId/reviewed` beside tour's
  (review.ts:1016-1034). Profile resolution rides along unused, tour-style.
- Pi mirror: in `apps/pi-extension/vendor.sh`, add `guide` to the flat shared-module
  list (line 10, next to `tour`) and duplicate the `tour-review` special block
  (lines 30-38) for `guide-review` (parent-relative + `@plannotator/shared/guide`
  import rewrites); hand-mirror the two routes + provider branches in
  `apps/pi-extension/server/serverReview.ts`.

## Client (`packages/review-editor`)

**State (App.tsx):**
- `guideOpen: boolean` beside `isFileTreeOpen` (App.tsx:216).
- `activeGuideJobId: string | null` ‚Äî latest completed `guide` job; completion
  observed by extending the tour auto-open effect pattern (App.tsx:780-792) with a ref
  `Set` dedupe. Auto-switch `guideOpen = true` on first completion (mirrors tour
  auto-open; confirm during build whether auto-takeover feels right vs. badge pulse).
- Takeover mechanics: **CSS-hide** the dock wrapper (App.tsx:2793) and file tree when
  `guideOpen`; never unmount `<DockviewReact>` (layout loss, proven by the
  files.length===0 branch). Guide screen renders as a peer branch in the main content
  row (App.tsx:2671+). Right sidebar untouched.
- Focus arbitration fix (verified simpler than spiked): `focusedFilePath` is computed
  once at ReviewStateContext population (`App.tsx:1778`, `files[activeFileIndex]`) and
  every dock panel derives `isFocused` from it (`ReviewDiffPanel.tsx:22`). Null it out
  there when `guideOpen` ‚Äî one line, and all CSS-hidden dock DiffViewers lose focus
  claims at the source. Guide-side instances arbitrate among themselves by section
  visibility.

**Header:** `Guide` badge immediately after the file-tree toggle divider
(App.tsx:2274), styled per the destination-dropdown pill (App.tsx:2334-2348); active
state while `guideOpen`. Shortcut `Mod+Shift+G` in `reviewEditorShortcuts`
(precedent: toggleTour, shortcuts.ts:58-76).

**New components (`components/guide/`):**
- `GuideScreen` ‚Äî takeover root; branches: empty state / generating / guide view.
- `GuideEmptyState` ‚Äî Notion-like page; engine/model/effort via a new
  `guide*` slice in `useAgentSettings` (cookie hook, mirrors tour slice) rendered with
  selector primitives promoted out of AgentsTab (`ConfigRow`, `SegmentedPicker`,
  `SelectMenu` ‚Üí shared file); Generate button calls the App-level `launchJob` from
  the single `useAgentJobs()` instance (no second SSE connection).
- `GuideGenerating` ‚Äî job status + live log peek via existing jobs/jobLogs props;
  cancel = existing killJob.
- `GuideView` ‚Äî title, meta, ordered `GuideSectionCard`s + trailing "Everything else"
  from `unplacedFiles`.
- `GuideSectionCard` ‚Äî title, `NN / MM`, Reviewed checkbox (checked ‚Üí collapses to a
  single row), overview (existing markdown rendering), then per `GuideDiffRef` a
  `GuideDiffSection`.
- `GuideDiffSection` ‚Äî thin `ReviewDiffPanel`-style adapter: resolves
  `files.find(f => f.path === ref.file)`, mounts one `DiffViewer` + `ToolbarHost`
  wired to the same `handleAddAnnotation`; missing file ‚Üí "outdated" chip (anchor
  drift degrades gracefully).
- `useGuideData` hook ‚Äî copy of `useTourData` shape: fetch `GET /api/guide/:jobId`,
  debounced `PUT .../reviewed` with keepalive flush; plus a demo guide
  (`demoGuide.ts`) behind the same dev toggle pattern as `DEMO_TOUR_ID`.

**AgentsTab:** add `guide` as a launchable mode (like tour) so the sidebar can also
launch/relaunch guides; guide jobs show in the job list; a completed guide job's card
gets an "Open guide" action ‚Üí `guideOpen`.

## Reviewed-state & staleness semantics

- Reviewed checkboxes are per-section (v1: no per-diff checkbox ‚Äî cut from wireframe
  to keep state model minimal; revisit if it feels lacking).
- Reviewed state keyed by jobId, in-memory (tour checklist precedent). Regenerating
  creates a new job ‚Üí fresh reviewed state, old guide still fetchable by old jobId.
- Diff staleness: guide renders whatever the *current* `files` array holds ‚Äî if the
  user refreshes/switches the diff after generation, refs that still resolve render
  the new content; refs that don't show the outdated chip and a "Regenerate guide"
  affordance. The guide never pins its own patch copy.

## Build order (each step shippable)

1. Shared types + server module + provider branches + routes (+ demo guide fixture).
2. Header badge + takeover branch + GuideScreen empty/generating states + launch
   (settings slice + promoted selector primitives).
3. GuideView/SectionCard/DiffSection with annotation wiring + reviewed persistence +
   focus-arbitration fix.
4. AgentsTab integration, Pi mirror, `bun run --cwd apps/review build && bun run
   build:hook` pipeline check, docs (CLAUDE.md endpoint tables).

## Out of scope (v1)

Durable guide persistence; multiple concurrent guides UI; hunk-level slicing;
guide sharing; marker engines (cursor/opencode) as guide generators.
